# Regional Capital Cost Differences

This folder contains the files and data used to create the regional capital cost differences (difference = multiplier - 1).
The multipliers are based on the city-based appendix tables of their report "Capital Cost Estimates for Utility Scale Electricity Generating Plants."
The current values are from the 2024 report at [https://www.eia.gov/analysis/studies/powerplants/capitalcost/pdf/capital_cost_AEO2025.pdf](https://www.eia.gov/analysis/studies/powerplants/capitalcost/pdf/capital_cost_AEO2025.pdf).

The city-based multipliers are entered into the first tab in the cc_scalefactor_city_v3.xlsx spreadsheet, and then output capital cost multipliers are automatically generated.  This spreadsheet was created by Donna Heimiller.  Outputs are produced for the supply curve points (SCgid), the resource regions (demreg), and the balancing area regions (BA).  Right now this process is manual (copy in the city based multipliers, copy out the resulting multipliers for ReEDS).

## Generating New Capital Cost Multipliers

New values can be generated by simply updating the city-based data for a given technology.  

The script reg_cap_cost_mult.py will create county-level capital cost differences based on the SCgid multipliers.

For additional information, including maps of the multipliers, see \\nrelnas01\ReEDS\_ReEDS Documentation\Regional Capital Cost Multipliers.

Questions about the data or processes in this folder can be directed to Wesley Cole.

**Steps for updating the regional capital cost multipliers in ReEDS:**

1. Get the most recent version of 'Capital Cost and Performance Characteristics for Utility-Scale Electric Power Generating Technologies'

2. Update location_based_cost_adjustments.csv with the most recent location based cost adjustments (can be found in the appendix of 'Capital Cost and Performance Characteristics for Utility-Scale Electric Power Generating Technologies')

3. For each technology, enter the following values into cc_scalefactor_city_v3.xlsx (making sure the states/cities match up): Base, Location Variation, DelatCostD
   - Add the output from the 'output_SCgid' tab into reg_cap_cost_mult.csv for the corresponding technology.

4. Once you have the multipliers, run reg_cap_cost_mult.py to get a new reg_cap_cost_diff_default.csv (you'll want to use the reeds2 conda environment).
Multipliers are converted to differences to avoid needing to add 1's for technologies with no multipliers, and to mitigate the effect of missing data;
in GAMS, empty data and zeros are equivalent, so we can leave out any techs with a multiplier of 1 (difference of 0).

**Summary of Files:**
- `cc_scalefactor_city_v3.xlsx`: Contains city-based multipliers and automatically generates output capital cost multipliers for supply curve points, resource regions, and balancing areas.
- `location_based_cost_adjustments.xlsx`: Contains location-based cost adjustment data from the EIA "Capital Cost and Performance Characteristics" report appendix.
- `reg_cap_cost_diff_default.csv`: Output file containing county-level capital cost differences (multiplier-1) for different technology categories. Used as input to the ReEDS model.
- `reg_cap_cost_mult.csv`: Input file containing capital cost multipliers for each supply curve point (sc_gid) and technology category, derived from 'cc_scalefactor_city_v3.xlsx'.
- `reg_cap_cost_mult.py`: Python script that processes supply curve multipliers and creates county-level capital cost differences by mapping supply curve points to counties and averaging multipliers within each county.
- `scgid_mapping.csv`: Contains a mapping of supply curve points to counties and BAs.
